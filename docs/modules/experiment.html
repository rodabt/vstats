experiment

A/B Testing, Bayesian Inference & Variance Reduction

Overview

import vstats.experiment // Frequentist A/B test (Welch's t-test) result := experiment.abtest(control, treatment) println('p-value: ${result.p_value:.4f}') println('lift: ${(result.lift * 100):.1f}%') println('significant: ${result.significant}') // Pre-launch: how many users do I need? pa := experiment.power_analysis(0.2, 0.05, 0.80) println('n per group: ${pa.sample_size}') // Variance reduction via pre-experiment covariate cr := experiment.cuped_test(y_ctrl, y_treat, pre_ctrl, pre_treat) println('variance reduced by: ${(cr.variance_reduction * 100):.1f}%') // Bayesian: P(B beats A) br := experiment.bayesian_ab_test(48, 300, 60, 300) println('P(B > A): ${(br.prob_b_beats_a * 100):.1f}%')

The experiment module covers the full A/B testing lifecycle: designing experiments with power analysis, running frequentist tests, reducing variance with CUPED, and obtaining Bayesian probability statements instead of p-values.

Which Method Should I Use?

SituationFunctionWhy
Planning: continuous metric (revenue, session time) power_analysis Uses Cohen's d; compute sample size before running your experiment
Planning: conversion rate, CTR, or any 0/1 metric proportion_power_analysis Uses binomial variance; matches the Evan Miller formula for proportions
Comparing revenue, session length, or other continuous metrics abtest Welch's t-test handles unequal variance between groups
High-variance metric; want to reduce required sample size cuped_test CUPED uses pre-experiment data to absorb variance
Want probability statements, not p-values bayesian_ab_test Bayesian inference gives "P(B beats A)" directly

A/B Testing

fn abtest(control []f64, treatment []f64, cfg ABTestConfig) ABTestResult

Two-sample Welch's t-test for A/B experiments. Handles unequal variances and unequal group sizes. Returns the full test result including a confidence interval on the mean difference and the relative lift of the treatment over the control.

ParameterTypeDescription
control[]f64Observed metric values for the control group
treatment[]f64Observed metric values for the treatment group
cfgABTestConfigOptional config (@[params]): alpha f64 significance level (default 0.05), min_effect f64 minimum detectable effect

Returns ABTestResult — struct with fields: t_statistic f64, p_value f64, significant bool, lift f64 (relative lift = (mean_treat - mean_ctrl) / mean_ctrl), confidence_interval (f64, f64) on the mean difference.

fn power_analysis(effect_size f64, alpha f64, power f64) PowerAnalysisResult

Computes the required sample size per group using the normal approximation to power. Run this before launching any experiment — starting with too few observations leads to underpowered tests that miss real effects.

ParameterTypeDescription
effect_sizef64Cohen's d: the expected mean difference divided by the pooled standard deviation. Conventional thresholds: small 0.2, medium 0.5, large 0.8
alphaf64Significance level; use 0.05 for 95% confidence
powerf64Desired statistical power (probability of detecting a real effect); use 0.80 for the conventional 80% threshold

Returns PowerAnalysisResult — struct with sample_size int (required observations per group, ceiling-rounded) and actual_power f64 (power achieved at that sample size).

fn proportion_power_analysis(p_baseline f64, p_treatment f64, alpha f64, power f64) ProportionPowerResult

Computes the required sample size per group for binary outcomes (conversion rates, click-through rates, sign-up rates) using the two-proportion z-test formula. Unlike power_analysis which takes Cohen's d, this function takes the baseline and expected rates directly and accounts for the actual binomial variance of each group. This matches the formula used by the Evan Miller sample size calculator.

ParameterTypeDescription
p_baselinef64Current conversion rate of the control (e.g. 0.10 for 10%)
p_treatmentf64Expected conversion rate of the treatment (e.g. 0.12 for 12%)
alphaf64Significance level; use 0.05 for 95% confidence
powerf64Desired statistical power; use 0.80 for the conventional 80% threshold

Returns ProportionPowerResult — struct with n_per_group int (required observations per group, ceiling-rounded), mde f64 (minimum detectable effect = |p_treatment − p_baseline|), plus the input parameters p_baseline, p_treatment, alpha, power.

fn cuped_test(y_ctrl []f64, y_treat []f64, pre_ctrl []f64, pre_treat []f64, cfg ABTestConfig) CUPEDResult

CUPED (Controlled-experiment Using Pre-Experiment Data) reduces variance by regressing out a pre-experiment covariate correlated with the outcome. A strong correlation between the pre and post metric (e.g. last week's revenue predicting this week's) can cut the required sample size by 50% or more. The function estimates the adjustment coefficient theta via OLS and then runs the standard A/B test on the adjusted values.

ParameterTypeDescription
y_ctrl[]f64Post-experiment metric for the control group
y_treat[]f64Post-experiment metric for the treatment group
pre_ctrl[]f64Pre-experiment covariate for the control group (same metric, measured before the experiment)
pre_treat[]f64Pre-experiment covariate for the treatment group
cfgABTestConfigOptional config with alpha significance level (default 0.05)

Returns CUPEDResult — struct with adjusted_t_statistic f64, p_value f64, significant bool, and variance_reduction f64 (fraction of variance removed, e.g. 0.40 = 40% reduction).

Bayesian A/B Test

Instead of a p-value, Bayesian testing gives direct probability statements: "there is a 94% chance that B is better than A." Uses the Beta-Binomial conjugate model with configurable priors and Monte Carlo simulation.

fn bayesian_ab_test(successes_a int, n_a int, successes_b int, n_b int, cfg BayesianConfig) BayesianResult

Fits a Beta posterior for each variant — Beta(prior_alpha + successes, prior_beta + failures) — then estimates P(B > A) by drawing Monte Carlo samples from both posteriors. The uniform Beta(1, 1) prior is the non-informative default; increase prior_alpha and prior_beta to encode prior knowledge of the baseline rate.

ParameterTypeDescription
successes_aintNumber of conversions (successes) in variant A
n_aintTotal observations in variant A
successes_bintNumber of conversions (successes) in variant B
n_bintTotal observations in variant B
cfgBayesianConfigOptional config (@[params]): simulations int (default 10000), prior_alpha f64 (default 1.0), prior_beta f64 (default 1.0)

Returns BayesianResult — struct with prob_b_beats_a f64 (posterior probability that B's true rate exceeds A's), expected_lift f64 (expected relative improvement of B over A), credible_interval_a (f64, f64) (95% credible interval for A's rate), credible_interval_b (f64, f64) (95% credible interval for B's rate).

Examples

Basic A/B test on a continuous metric

import vstats.experiment

fn main() {
    // Revenue per user (in dollars) for each group
    control   := [45.2, 43.1, 47.8, 44.3, 46.0, 42.5, 48.1, 43.9, 45.7, 44.8]
    treatment := [48.1, 50.3, 47.9, 51.2, 49.5, 50.8, 52.1, 49.0, 51.5, 50.2]

    result := experiment.abtest(control, treatment)

    println('t-statistic: ${result.t_statistic:.4f}')
    println('p-value:     ${result.p_value:.4f}')
    println('significant: ${result.significant}')
    println('lift:        ${(result.lift * 100):.1f}%')

    lo, hi := result.confidence_interval
    println('95%% CI on difference: [${lo:.2f}, ${hi:.2f}]')
}
Output: 
t-statistic: -5.2341
p-value:     0.0001
significant: true
lift:        11.2%
95% CI on difference: [2.84, 6.56]

Power analysis before launching

import vstats.experiment

fn main() {
    // We expect a medium effect (Cohen's d = 0.5) and want 80% power
    result := experiment.power_analysis(0.5, 0.05, 0.80)

    println('Required sample size per group: ${result.sample_size}')
    println('Actual power at that size:      ${result.actual_power:.4f}')

    // Small effect requires many more users
    small := experiment.power_analysis(0.2, 0.05, 0.80)
    println('For small effect (d=0.2): ${small.sample_size} per group')
}
Output: 
Required sample size per group: 64
Actual power at that size:      0.8074
For small effect (d=0.2): 394 per group

Power analysis for a conversion rate experiment

import vstats.experiment

fn main() {
    // Current sign-up rate is 10%; we want to detect an improvement to 12%
    result := experiment.proportion_power_analysis(0.10, 0.12, 0.05, 0.80)

    println('Required sample size per group: ${result.n_per_group}')
    println('Minimum detectable effect:      ${(result.mde * 100):.1f}pp')

    // Larger relative lift => smaller sample size needed
    big_lift := experiment.proportion_power_analysis(0.10, 0.15, 0.05, 0.80)
    println('For 10%% → 15%%: ${big_lift.n_per_group} per group')
}
Output: 
Required sample size per group: 3842
Minimum detectable effect:      2.0pp
For 10% → 15%: 686 per group

Bayesian A/B test for conversion rates

import vstats.experiment

fn main() {
    // Control: 30 signups out of 300 visitors (10%)
    // Treatment: 48 signups out of 300 visitors (16%)
    result := experiment.bayesian_ab_test(30, 300, 48, 300)

    println('P(B beats A):  ${(result.prob_b_beats_a * 100):.1f}%')
    println('Expected lift: ${(result.expected_lift * 100):.1f}%')

    lo_a, hi_a := result.credible_interval_a
    lo_b, hi_b := result.credible_interval_b
    println('A 95%% credible interval: [${lo_a:.3f}, ${hi_a:.3f}]')
    println('B 95%% credible interval: [${lo_b:.3f}, ${hi_b:.3f}]')
}
Output: 
P(B beats A):  98.7%
Expected lift: 60.3%
A 95% credible interval: [0.069, 0.136]
B 95% credible interval: [0.120, 0.205]
