experiment

A/B Testing, Causal Inference & Experimentation

Overview

The experiment module provides tools for designing and analyzing controlled experiments. It covers the full lifecycle: calculating required sample size before launch, running statistical tests after the experiment, and applying variance reduction or causal inference methods.

Import with: import vstats.experiment

Which Method Should I Use?

Situation Function Why
Planning: how many users do I need? sample_size_proportions / sample_size_means Always compute sample size before running your experiment
Comparing click-through, signup, or purchase rates proportion_test Binary outcomes need a z-test, not a t-test
Comparing revenue, session length, or other continuous metrics abtest Welch's t-test handles unequal variance between groups
Need to check results early without inflating false positives sprt_test SPRT gives statistically valid early-stopping decisions
Want probability statements, not p-values bayesian_ab_test Bayesian inference gives "P(B beats A)" directly
High-variance metric; want to reduce required sample size cuped_test CUPED uses pre-experiment data to absorb variance
Treatment was not randomly assigned; observational data PSM / DiD Propensity matching and difference-in-differences adjust for confounders

Sample Size Calculators

Always calculate your required sample size before running an experiment. Starting with too few observations leads to underpowered tests that miss real effects.

fn sample_size_proportions(baseline_rate f64, mde f64, alpha f64, power f64) SampleSizeResult

Computes required observations per group for a binary metric (clicks, signups, purchases).

ParameterTypeDescription
baseline_ratef64Your current conversion rate, e.g. 0.05 for 5%
mdef64Minimum detectable effect as an absolute change, e.g. 0.01 to detect a 5% → 6% lift
alphaf64Significance level. Use 0.05 for 95% confidence
powerf64Desired statistical power. Use 0.80 (80% chance of detecting a real effect)
result := experiment.sample_size_proportions(0.05, 0.01, 0.05, 0.80)
println(result.n_per_group)  // ~8100 — needed per group
println(result.total_n)      // ~16200 — total experiment size

fn sample_size_means(baseline_mean f64, baseline_std f64, mde_absolute f64, alpha f64, power f64) SampleSizeResult

Computes required observations per group for a continuous metric (revenue, latency, engagement).

ParameterTypeDescription
baseline_meanf64Current average value
baseline_stdf64Current standard deviation (estimate from historical data)
mde_absolutef64Absolute change in mean to detect, e.g. 5.0 to detect a $5 revenue lift
alphaf64Significance level
powerf64Desired power
result := experiment.sample_size_means(45.0, 20.0, 3.0, 0.05, 0.80)
println(result.n_per_group)  // observations per group
println(result.effect_size)  // Cohen's d = mde / std

struct SampleSizeResult

FieldTypeDescription
n_per_groupintRequired observations per group (ceiling-rounded)
total_nintn_per_group * 2
alphaf64Significance level used
powerf64Power used
mdef64MDE used
baselinef64Baseline rate or mean used
effect_sizef64Implied effect size (Cohen's d for means; h-style for proportions)
baseline_stdf64Std dev used (means only; 0 for proportions)
methodstring"proportions" or "means"

Proportion Test

Use this when your outcome is binary: each user either converted or didn't. This is the correct test for click-through rates, signup rates, purchase rates, etc.

fn proportion_test(successes_a int, n_a int, successes_b int, n_b int, cfg ProportionTestConfig) ProportionTestResult

Two-proportion z-test using a pooled standard error under H₀. The confidence interval uses the unpooled standard error.

// 30 conversions / 300 users vs 48 conversions / 300 users
result := experiment.proportion_test(30, 300, 48, 300)
println(result.rate_a)        // 0.10 (10%)
println(result.rate_b)        // 0.16 (16%)
println(result.relative_lift) // 0.60 (60% relative lift)
println(result.p_value)       // e.g. 0.012
println(result.significant)   // true
println('${(result.diff * 100):.1f}% ± ${((result.ci_upper - result.ci_lower) / 2 * 100):.1f}pp')

struct ProportionTestResult

FieldDescription
rate_a / rate_bObserved conversion rates
diffrate_b - rate_a (absolute difference)
relative_liftdiff / rate_a
z_statisticTest statistic (pooled SE)
p_valueTwo-tailed p-value
significantWhether p_value < alpha
ci_lower / ci_upperConfidence interval on the difference (unpooled SE)
pooled_seStandard error under H₀ (used for z-statistic)
n_a / n_bGroup sizes

A/B Test — Continuous Metrics

fn abtest(control []f64, treatment []f64, cfg ABTestConfig) ABTestResult

Welch's t-test for continuous metrics. Handles unequal variances and unequal group sizes. Returns Cohen's d effect size, confidence interval on the mean difference, and relative lift.

control   := [45.2, 43.1, 47.8, 44.3, 46.0]  // revenue per user
treatment := [48.1, 50.3, 47.9, 51.2, 49.5]
result := experiment.abtest(control, treatment)
println(result.p_value)      // e.g. 0.031
println(result.relative_lift) // e.g. 0.092 (9.2% lift)
println(result.ci_lower)     // lower bound of CI on difference

ABTestConfig fields: alpha f64 = 0.05

Sequential Testing (SPRT)

The SPRT lets you check experiment results at any point without inflating the false positive rate. Call it repeatedly with cumulative totals as data comes in.

fn sprt_test(successes_a int, n_a int, successes_b int, n_b int, cfg SPRTConfig) SPRTResult

Stateless function — pass cumulative totals each time. Stop the experiment when decision is not .continue_testing.

cfg := experiment.SPRTConfig{ alpha: 0.05, beta: 0.20, mde: 0.02 }

// Check on day 3 (cumulative totals so far)
r := experiment.sprt_test(120, 2400, 145, 2400, cfg)
if r.decision == .reject_null {
    println('Significant! Stop the experiment.')
} else if r.decision == .accept_null {
    println('Futile. The effect is smaller than your MDE.')
} else {
    println('Keep collecting data.')
}

struct SPRTConfig

Not a @[params] struct — you must construct it explicitly because mde has no sensible default.

FieldDefaultDescription
alpha0.05Max acceptable false positive rate
beta0.20Max acceptable false negative rate (1 - power)
mderequiredMinimum detectable effect (absolute rate difference)

enum SPRTDecision

ValueMeaning
.continue_testingNot enough evidence yet — keep running
.reject_nullSignificant effect detected — safe to stop
.accept_nullEffect is below MDE — futile to continue

Bayesian A/B Test

Instead of a p-value, Bayesian testing gives you direct probability statements: "there is a 94% chance that B is better than A." Uses the Beta-Binomial conjugate model with configurable priors.

fn bayesian_ab_test(successes_a int, n_a int, successes_b int, n_b int, cfg BayesianConfig) BayesianResult 

result := experiment.bayesian_ab_test(30, 300, 48, 300)
println('P(B beats A): ${(result.prob_b_beats_a * 100):.1f}%')
println('Expected loss if we ship B: ${result.expected_loss_b:.4f}')
println('A credible interval: [${result.ci_lower_a:.3f}, ${result.ci_upper_a:.3f}]')
println('B credible interval: [${result.ci_lower_b:.3f}, ${result.ci_upper_b:.3f}]')

Interpreting expected loss: expected_loss_b is the average rate you "give up" by shipping B if A was actually better. A decision rule: ship B if expected_loss_b < threshold (e.g. 0.001).

struct BayesianConfig (optional, @[params])

FieldDefaultDescription
alpha_prior1.0Beta prior shape — set higher to encode prior knowledge of the success rate
beta_prior1.0Beta prior shape — Beta(1,1) is the uniform (non-informative) prior
n_samples10000Monte Carlo samples — increase for more precise estimates

CUPED — Variance Reduction

fn cuped_test(y_ctrl []f64, y_treat []f64, pre_ctrl []f64, pre_treat []f64, cfg ABTestConfig) CUPEDResult

CUPED (Controlled-experiment Using Pre-Experiment Data) reduces variance by adjusting outcomes using a correlated pre-experiment covariate. A strong correlation (e.g. last week's revenue predicts this week's revenue) can cut required sample size by 50% or more.

Pass the same pre-experiment metric as pre_ctrl / pre_treat (e.g. revenue in the week before the experiment). The function estimates the adjustment coefficient theta and runs the A/B test on adjusted values.

result := experiment.cuped_test(y_control, y_treatment, pre_control, pre_treatment)
println('Variance reduced by: ${(result.variance_reduction * 100):.1f}%')
println('Adjusted p-value: ${result.adjusted_result.p_value:.4f}')

Causal Inference

Use these when treatment was not randomly assigned (observational data).

Propensity Score Matching (PSM)

Matches treated and control units on the probability of receiving treatment, controlling for selection bias.

ps_model := experiment.estimate_propensity_scores(x_covariates, treatment)
matching := experiment.match_nearest_neighbor(ps_model)
balance  := experiment.check_balance(x_covariates, treatment, matching)
ate      := experiment.ate_matched(y_outcomes, treatment, matching)

Difference-in-Differences (DiD)

Estimates causal effects using pre/post data across treated and control groups, assuming parallel pre-period trends.

// Simple 2x2 DiD
result := experiment.did_2x2(y_treat_pre, y_treat_post, y_ctrl_pre, y_ctrl_post)
println('DiD effect: ${result.did_effect:.4f}')

// Test whether parallel trends assumption holds
pt := experiment.test_parallel_trends(y_treated_pre, y_control_pre, time_pre)
println('Parallel trends hold: ${pt.parallel_trends_hold}')
