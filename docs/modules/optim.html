optim

Gradient-Based Optimization — numerical differentiation and gradient descent primitives

Overview

import vstats.optim // Numerical gradient of any scalar function f := fn(v []f64) f64 { return v[0]*v[0] + v[1]*v[1] } grad := optim.gradient(f, [3.0, 4.0], 0.0001) // [-6.0, -8.0] approx // Gradient descent loop mut v := [10.0, 10.0] for _ in 0..100 { g := optim.gradient(f, v, 0.0001) v = optim.gradient_step(v, g, -0.1) // step in negative gradient direction }

The optim module provides building blocks for gradient descent. Gradients are computed numerically via finite differences — no symbolic differentiation required. Combine gradient and gradient_step in a loop to minimize any differentiable scalar function.

API Reference

fn difference_quotient(f fn(f64) f64, x f64, h f64) f64

Numerical derivative of a univariate function at point x using the forward difference quotient: (f(x + h) - f(x)) / h. Use a small h (e.g. 0.0001) for a good approximation.

ParameterTypeDescription
ffn(f64) f64Scalar function of one variable to differentiate
xf64Point at which to evaluate the derivative
hf64Step size for the finite difference; smaller values give better approximations but risk numerical instability

Returns f64 — approximate derivative f′(x).

fn partial_difference_quotient(f fn([]f64) f64, v []f64, i int, h f64) f64

Numerical partial derivative of a multivariate function with respect to the i-th component of v. Perturbs only v[i] by h, leaving all other components unchanged.

ParameterTypeDescription
ffn([]f64) f64Scalar function of a vector argument
v[]f64Point at which to evaluate the partial derivative
iintIndex of the component to differentiate with respect to
hf64Step size for the finite difference

Returns f64 — approximate partial derivative ∂f/∂vᵢ at v.

fn gradient(f fn([]f64) f64, v []f64, h f64) []f64

Numerical gradient of a multivariate scalar function at point v. Computes the partial derivative with respect to each component using partial_difference_quotient. The gradient vector points in the direction of steepest ascent.

ParameterTypeDescription
ffn([]f64) f64Scalar function of a vector argument to differentiate
v[]f64Point at which to evaluate the gradient
hf64Step size for finite differences; typically 0.0001

Returns []f64 — gradient vector ∇f(v) of the same length as v.

fn gradient_step[T](v []T, gradient_vector []T, step_size T) []T

Applies one step of gradient descent (or ascent). Computes v + step_size × gradient_vector element-wise. Pass a negative step_size (learning rate) to descend; pass a positive value to ascend.

ParameterTypeDescription
v[]TCurrent parameter vector
gradient_vector[]TGradient at v; must be the same length as v
step_sizeTScalar multiplier (learning rate). Use a negative value to move against the gradient (minimization)

Returns []T — updated parameter vector of the same length as v.

fn sum_of_squares_gradient[T](v []T) []T

Closed-form gradient of the sum-of-squares function f(v) = Σ vᵢ². The gradient is simply 2v — each component doubled. Useful as a reference function for testing optimization routines and understanding gradient descent convergence.

ParameterTypeDescription
v[]TInput vector

Returns []T — gradient vector where each element is 2 × v[i].

Examples

Numerical derivative

import vstats.optim

fn main() {
    // Derivative of f(x) = x^2 at x = 3 should be 6.0
    f := fn(x f64) f64 { return x * x }
    dfdx := optim.difference_quotient(f, 3.0, 0.0001)
    println("f'(3) ≈ ${dfdx:.4f}")  // expect ~6.0
}
Output: 
f'(3) ≈ 6.0001

Minimizing a function with gradient descent

import vstats.optim
import vstats.linalg

fn main() {
    // Minimize f(v) = sum of squares; minimum is at v = [0, 0, 0]
    f := fn(v []f64) f64 {
        return v[0]*v[0] + v[1]*v[1] + v[2]*v[2]
    }

    mut v := [10.0, 10.0, 10.0]
    for _ in 0..200 {
        grad := optim.gradient(f, v, 0.0001)
        v = optim.gradient_step(v, grad, -0.1)
    }

    println("Result: ${v}")
    println("f(v) = ${f(v):.6f}")
}
Output: 
Result: [0.000018, 0.000018, 0.000018]
f(v) = 0.000000

Using the closed-form gradient

import vstats.optim

fn main() {
    // sum_of_squares_gradient is faster than numerical gradient
    // when the analytical form is known
    mut v := [5.0, -3.0, 2.0]
    for _ in 0..100 {
        grad := optim.sum_of_squares_gradient(v)
        v = optim.gradient_step(v, grad, -0.1)
    }
    println("v ≈ ${v}")
}
Output: 
v ≈ [0.000003, -0.000002, 0.000001]
