nn

Neural Networks — fully-connected layers, activation functions, convolution, pooling, and a comprehensive loss function library.

Overview

The nn module provides the building blocks for constructing and training feedforward neural networks. The highest-level entry point is sequential, which creates a fully-connected network from a list of layer sizes in one call. Lower-level primitives — DenseLayer, ActivationLayer, BatchNormLayer, dropout, convolution, and pooling — are also exposed for custom architectures.

Training is driven by mini-batch gradient descent via train. A parallel variant (train_parallel) averages gradients across worker threads for larger workloads. Predictions can be made per-sample with predict_single or in batch with predict.

Quick start

import vstats.nn

fn main() {
    // Build a 3-8-1 network with ReLU hidden activations
    mut net := nn.sequential([3, 8, 1], 'relu')

    x_train := [
        [0.2, 0.5, 0.1],
        [0.9, 0.3, 0.7],
        [0.4, 0.8, 0.2],
        [0.6, 0.1, 0.9],
    ]
    y_train := [0.0, 1.0, 0.0, 1.0]

    config := nn.training_config(0.01, 200, 2)
    net.train(x_train, y_train, config)

    preds := net.predict(x_train)
    for p in preds {
        println(p)
    }
    // [0.0621]
    // [0.9314]
    // [0.0487]
    // [0.9178]
}

Network

High-level training and inference API. All functions live in nn/network.v.

fn sequential(layer_sizes []int, activation_fn string) NeuralNetwork

Creates a fully-connected feedforward network. Each adjacent pair in layer_sizes becomes one DenseLayer followed by the chosen activation. The last layer does not have an activation applied by default, so it can be used for both regression and classification (add a softmax call on predictions for multi-class output).

ParameterTypeDescription
layer_sizes[]intSizes of each layer including input and output, e.g. [4, 16, 8, 1]
activation_fnstringHidden-layer activation: "relu", "sigmoid", or "tanh"

Returns NeuralNetwork — initialized network with Xavier-weighted layers ready for training.

fn training_config(lr f64, epochs int, batch_size int) TrainingConfig

Constructs a TrainingConfig struct with explicit hyperparameters.

ParameterTypeDescription
lrf64Learning rate for gradient descent
epochsintNumber of full passes over the training data
batch_sizeintNumber of samples per mini-batch

Returns TrainingConfig — configuration struct to pass to train.

fn default_training_config() TrainingConfig

Returns a sensible default TrainingConfig: learning rate 0.01, 100 epochs, batch size 32. Use this as a starting point and override fields as needed.

Returns TrainingConfig — default hyperparameter configuration.

fn (mut net NeuralNetwork) train(x_train [][]f64, y_train []f64, config TrainingConfig)

Trains the network in place using mini-batch gradient descent. Samples are shuffled each epoch. Gradients are averaged within each mini-batch before the weight update.

ParameterTypeDescription
x_train[][]f64Training feature matrix; each inner slice is one sample
y_train[]f64Training targets; must have the same length as x_train
configTrainingConfigHyperparameters (learning rate, epochs, batch size)

Returns nothing — modifies the network weights in place.

fn (mut net NeuralNetwork) train_parallel(x_train [][]f64, y_train []f64, config TrainingConfig)

Multi-threaded variant of train. The mini-batch is split across worker threads; gradients are averaged before the weight update. Useful when batch sizes are large enough to saturate a single core.

ParameterTypeDescription
x_train[][]f64Training feature matrix
y_train[]f64Training targets
configTrainingConfigHyperparameters passed to each worker thread

Returns nothing — modifies the network weights in place.

fn (mut net NeuralNetwork) predict(x [][]f64) [][]f64

Runs a forward pass for every sample in the batch and returns a slice of output vectors.

ParameterTypeDescription
x[][]f64Input batch; each inner slice is one sample

Returns [][]f64 — one output vector per sample, in the same order as the input.

fn (mut net NeuralNetwork) predict_single(x []f64) []f64

Convenience wrapper for running a single-sample forward pass without allocating a batch wrapper.

ParameterTypeDescription
x[]f64A single input sample

Returns []f64 — output activations for the single sample.

fn (mut net NeuralNetwork) predict_parallel(x [][]f64) [][]f64

Parallel batch prediction that distributes samples across worker threads. Drop-in replacement for predict when inference latency matters.

ParameterTypeDescription
x[][]f64Input batch

Returns [][]f64 — output vectors in the same order as the input.

fn (mut net NeuralNetwork) evaluate(x_test [][]f64, y_test []f64) f64

Computes mean squared error over a held-out test set without modifying network weights. Useful for monitoring generalization during or after training.

ParameterTypeDescription
x_test[][]f64Test feature matrix
y_test[]f64Ground-truth targets for the test set

Returns f64 — mean squared error across all test samples.

fn (mut net NeuralNetwork) forward(input []f64) []f64

Low-level single-sample forward pass. Sequentially applies each layer's forward method. Primarily used internally by train and predict_single, but exposed for custom training loops.

ParameterTypeDescription
input[]f64One input sample; length must match the network's input size

Returns []f64 — final layer output activations.

fn (mut net NeuralNetwork) backward(grad_output []f64, input []f64, learning_rate f64) []f64

Backpropagates a loss gradient through the entire network, updating all layer weights and biases immediately (online update). Use backward_with_accumulation for momentum-based updates.

ParameterTypeDescription
grad_output[]f64Loss gradient with respect to the network output
input[]f64The original input sample used in the corresponding forward pass
learning_ratef64Step size for the weight update

Returns []f64 — gradient with respect to the network input (useful when chaining networks).

fn (mut net NeuralNetwork) backward_with_accumulation(grad_output []f64, input []f64, learning_rate f64, momentum f64) []f64

Backpropagation with momentum-based gradient accumulation. Weight velocities are maintained per layer and updated as v = momentum * v - lr * grad before applying to the weight. A momentum value of 0.9 is typical.

ParameterTypeDescription
grad_output[]f64Loss gradient with respect to the network output
input[]f64The original input sample used in the forward pass
learning_ratef64Step size for the weight update
momentumf64Momentum coefficient, typically 0.9; 0.0 disables momentum

Returns []f64 — gradient with respect to the network input.

fn (net NeuralNetwork) get_weights() [][][]f64

Returns a copy of all layer weight matrices. The outer index is the layer, the next is the output neuron, and the innermost is the input neuron (row-major, [output][input]).

Returns [][][]f64 — weight tensor for all layers.

fn (net NeuralNetwork) get_biases() [][][]f64

Returns a copy of all layer bias vectors. Indexed as [layer][neuron].

Returns [][][]f64 — bias tensor for all layers.

fn (mut net NeuralNetwork) set_weights(weights [][][]f64)

Replaces all layer weight matrices. The shape must match the network architecture exactly. Primarily used for loading saved model weights or for weight-sharing between networks.

ParameterTypeDescription
weights[][][]f64New weights indexed as [layer][output_neuron][input_neuron]

Returns nothing — modifies the network in place.

Layers & Activations

Individual layer constructors and activation functions from nn/layers.v.

fn dense_layer(input_size int, output_size int) DenseLayer

Creates a fully-connected layer. Weights are initialized with Xavier uniform initialization (scale 1/sqrt(input_size)) and biases are set to 0.01. Pre-allocated output and gradient buffers avoid repeated heap allocations during training.

ParameterTypeDescription
input_sizeintNumber of input features
output_sizeintNumber of output neurons

Returns DenseLayer — initialized dense layer.

fn (mut layer DenseLayer) forward(input []f64) []f64

Computes output = W·input + b using cache-efficient row-major weight access. The result is written into a pre-allocated output_buffer — the returned slice references that buffer. If you need to keep the result across a second forward call, copy it first (e.g. result := layer.forward(x).clone()).

ParameterTypeDescription
input[]f64Input vector; length must equal input_size

Returns []f64 — pre-activation output (reference to internal buffer).

fn (mut layer DenseLayer) backward(grad_output []f64, input []f64, learning_rate f64, momentum f64) []f64

Computes input gradient and updates weights and biases with momentum SGD. Velocity is maintained per parameter: v = momentum * v - lr * grad, then w += v. Pass momentum = 0.0 for vanilla SGD.

ParameterTypeDescription
grad_output[]f64Gradient flowing back from the next layer
input[]f64The input that was fed into this layer's forward pass
learning_ratef64Step size
momentumf64Momentum coefficient; 0.0 = standard SGD

Returns []f64 — gradient with respect to the layer input (for the preceding layer).

fn activation_layer(activation_fn string) ActivationLayer

Creates a stateless activation layer that stores function pointers for the forward activation and its derivative. Falls back to ReLU for unrecognized names.

ParameterTypeDescription
activation_fnstringOne of "relu", "sigmoid", or "tanh"

Returns ActivationLayer — layer wrapping the chosen activation and its derivative.

fn (layer ActivationLayer) forward(input []f64) []f64

Applies the stored activation function element-wise to each value in input.

ParameterTypeDescription
input[]f64Pre-activation values

Returns []f64 — post-activation values, same length as input.

fn (layer ActivationLayer) backward(grad_output []f64, input []f64) []f64

Applies the stored activation derivative element-wise and multiplies by the incoming gradient (chain rule).

ParameterTypeDescription
grad_output[]f64Gradient from the next layer
input[]f64The pre-activation input used in the forward pass

Returns []f64 — gradient with respect to the pre-activation input.

fn batch_norm_layer(input_size int) BatchNormLayer

Creates a batch normalization layer. Initial gamma (scale) is 1.0 and beta (shift) is 0.0 per feature. Epsilon is 1e-5 to prevent division by zero.

ParameterTypeDescription
input_sizeintNumber of features to normalize

Returns BatchNormLayer — initialized batch norm layer.

fn (layer BatchNormLayer) forward(input []f64) []f64

Normalizes each feature to zero mean and unit variance using the layer's stored running statistics, then applies the learnable scale (gamma) and shift (beta): output[i] = gamma[i] * (input[i] - mean[i]) / sqrt(variance[i] + epsilon) + beta[i].

ParameterTypeDescription
input[]f64Input feature vector; length must equal input_size

Returns []f64 — normalized and scaled output.

fn relu(x f64) f64

Rectified Linear Unit activation. Returns x when positive, zero otherwise. The most commonly used hidden-layer activation because it avoids the vanishing gradient problem and is cheap to compute.

ParameterTypeDescription
xf64Input scalar

Returns f64 — max(0, x).

fn relu_derivative(x f64) f64

Derivative of ReLU. Returns 1.0 for positive inputs and 0.0 otherwise (subgradient at zero is 0).

ParameterTypeDescription
xf64Pre-activation scalar

Returns f64 — 1.0 if x > 0, else 0.0.

fn sigmoid(x f64) f64

Logistic sigmoid activation: 1 / (1 + e^-x). Maps any real number to the open interval (0, 1). Suitable for binary classification output layers. Delegates to utils.sigmoid.

ParameterTypeDescription
xf64Input scalar

Returns f64 — value in (0, 1).

fn sigmoid_derivative(x f64) f64

Derivative of the sigmoid function: sigmoid(x) * (1 - sigmoid(x)). Delegates to utils.sigmoid_derivative.

ParameterTypeDescription
xf64Pre-activation scalar

Returns f64 — derivative of sigmoid at x.

fn tanh(x f64) f64

Hyperbolic tangent activation. Maps any real number to (-1, 1), making it zero-centered unlike sigmoid. Often outperforms sigmoid in hidden layers.

ParameterTypeDescription
xf64Input scalar

Returns f64 — hyperbolic tangent of x, in (-1, 1).

fn tanh_derivative(x f64) f64

Derivative of tanh: 1 - tanh(x)^2. Peaks at 1.0 when x = 0 and approaches 0 in the tails.

ParameterTypeDescription
xf64Pre-activation scalar

Returns f64 — derivative of tanh at x.

fn softmax(x []f64) []f64

Converts a vector of raw scores into a probability distribution. Each element is exp(x[i] - max(x)) / sum(exp(x[j] - max(x))). Subtracting the maximum before exponentiating prevents overflow. Output elements sum to 1.0.

ParameterTypeDescription
x[]f64Raw logit scores, one per class

Returns []f64 — probability vector summing to 1.0.

fn dropout(input []f64, dropout_rate f64) []f64

Applies inverted dropout regularization. Each element is zeroed independently with probability dropout_rate. Surviving elements are scaled by 1 / (1 - dropout_rate) so the expected magnitude is preserved at inference. Passing dropout_rate <= 0 returns a clone of the input unchanged.

ParameterTypeDescription
input[]f64Activation vector to apply dropout to
dropout_ratef64Fraction of elements to zero out, in [0, 1)

Returns []f64 — masked and scaled activation vector.

fn flatten(data [][]f64) []f64

Converts a 2D slice into a contiguous 1D slice by concatenating rows in order. Delegates to linalg.flatten. Useful before passing image or time-series data into a dense layer.

ParameterTypeDescription
data[][]f642D input matrix

Returns []f64 — 1D concatenation of all rows.

fn reshape(data []f64, rows int, cols int) [][]f64

Reshapes a flat 1D slice into a 2D matrix using row-major order. Asserts that data.len == rows * cols.

ParameterTypeDescription
data[]f64Source 1D slice
rowsintNumber of rows in the output matrix
colsintNumber of columns in the output matrix

Returns [][]f64 — 2D matrix of shape [rows][cols].

Convolution & Pooling

1D signal-processing operations from nn/layers.v. These are useful for sequence models and 1D time-series inputs.

fn conv1d(input []f64, kernel []f64, stride int) []f64

Performs a 1D discrete convolution (cross-correlation) without padding. Each output position is the dot product of the kernel with a window of the input. Output length is (len(input) - len(kernel)) / stride + 1.

ParameterTypeDescription
input[]f641D input signal
kernel[]f64Convolutional filter weights
strideintStep size between successive windows

Returns []f64 — convolved output of length (len(input) - len(kernel)) / stride + 1.

fn max_pool1d(input []f64, pool_size int, stride int) []f64

Applies 1D max pooling. For each window of length pool_size, selects the maximum value. Reduces the sequence length while retaining dominant features. Output length is (len(input) - pool_size) / stride + 1.

ParameterTypeDescription
input[]f641D input signal
pool_sizeintWidth of the pooling window
strideintStep size between successive windows

Returns []f64 — downsampled output.

fn avg_pool1d(input []f64, pool_size int, stride int) []f64

Applies 1D average pooling. For each window of length pool_size, computes the arithmetic mean. Smoother than max pooling and less sensitive to noise spikes. Output length is (len(input) - pool_size) / stride + 1.

ParameterTypeDescription
input[]f641D input signal
pool_sizeintWidth of the pooling window
strideintStep size between successive windows

Returns []f64 — downsampled output.

Loss Functions

All loss functions live in nn/loss.v. Generic variants accept any numeric type T; non-generic variants operate on []f64 directly.

fn mse_loss[T](y_true []T, y_pred []T) f64

Mean Squared Error: average of squared differences between predictions and targets. The standard loss for regression. Delegates to utils.mse.

ParameterTypeDescription
y_true[]TGround-truth targets
y_pred[]TModel predictions, same length as y_true

Returns f64 — mean of (y_pred[i] - y_true[i])^2.

fn mse_loss_gradient[T](y_true []T, y_pred []T) []f64

Gradient of MSE with respect to predictions: 2 * (y_pred[i] - y_true[i]) / n. Pass directly to backward to train with MSE.

ParameterTypeDescription
y_true[]TGround-truth targets
y_pred[]TModel predictions

Returns []f64 — gradient vector of length n.

fn mae_loss[T](y_true []T, y_pred []T) f64

Mean Absolute Error: average of absolute differences. More robust to outliers than MSE because large errors are not squared. Delegates to utils.mae.

ParameterTypeDescription
y_true[]TGround-truth targets
y_pred[]TModel predictions

Returns f64 — mean of |y_pred[i] - y_true[i]|.

fn mae_loss_gradient[T](y_true []T, y_pred []T) []f64

Gradient of MAE: sign(y_pred[i] - y_true[i]) / n. Constant magnitude means learning does not accelerate for large errors, which can stabilize training.

ParameterTypeDescription
y_true[]TGround-truth targets
y_pred[]TModel predictions

Returns []f64 — gradient vector of length n.

fn binary_crossentropy_loss(y_true []f64, y_pred []f64) f64

Binary cross-entropy (log loss) for binary classification. Predictions are clipped to [1e-7, 1-1e-7] to prevent log(0). y_pred values should be sigmoid probabilities in (0, 1).

ParameterTypeDescription
y_true[]f64Binary labels in {0.0, 1.0}
y_pred[]f64Predicted probabilities from a sigmoid output

Returns f64 — mean binary cross-entropy loss.

fn binary_crossentropy_loss_gradient(y_true []f64, y_pred []f64) []f64

Gradient of binary cross-entropy with respect to y_pred: (y_pred[i] - y_true[i]) / (y_pred[i] * (1 - y_pred[i]) * n). Predictions are clipped to avoid division by zero.

ParameterTypeDescription
y_true[]f64Binary labels
y_pred[]f64Predicted sigmoid probabilities

Returns []f64 — gradient vector of length n.

fn categorical_crossentropy_loss(y_true [][]f64, y_pred [][]f64) f64

Categorical cross-entropy for multi-class classification. Both y_true and y_pred are batches of vectors. y_true must be one-hot encoded. y_pred should be softmax probabilities. Predictions are clipped to avoid log(0).

ParameterTypeDescription
y_true[][]f64One-hot encoded ground-truth labels; shape [batch][num_classes]
y_pred[][]f64Softmax probability vectors; shape [batch][num_classes]

Returns f64 — mean categorical cross-entropy over the batch.

fn sparse_categorical_crossentropy_loss(y_true []int, y_pred [][]f64) f64

Like categorical_crossentropy_loss but accepts integer class indices instead of one-hot vectors, avoiding the need to explicitly encode labels.

ParameterTypeDescription
y_true[]intInteger class indices, one per sample
y_pred[][]f64Softmax probability vectors; shape [batch][num_classes]

Returns f64 — mean categorical cross-entropy over the batch.

fn huber_loss[T](y_true []T, y_pred []T, delta f64) f64

Huber loss combines MSE for small errors and MAE for large errors, controlled by the threshold delta. For |error| <= delta: 0.5 * error^2; for |error| > delta: delta * (|error| - 0.5 * delta). Less sensitive to outliers than pure MSE while remaining smooth near zero.

ParameterTypeDescription
y_true[]TGround-truth targets
y_pred[]TModel predictions
deltaf64Threshold separating the MSE and MAE regimes; typically 1.0

Returns f64 — mean Huber loss.

fn hinge_loss[T](y_true []T, y_pred []T) f64

Hinge loss for SVM-style binary classifiers. Computes mean(max(0, 1 - y_true[i] * y_pred[i])). Labels must be -1 or +1; predictions are raw scores (not probabilities).

ParameterTypeDescription
y_true[]TLabels in {-1, +1}
y_pred[]TRaw decision scores

Returns f64 — mean hinge loss.

fn squared_hinge_loss[T](y_true []T, y_pred []T) f64

Squared hinge loss: mean(max(0, 1 - y_true[i] * y_pred[i])^2). Penalizes margin violations more strongly than standard hinge loss, leading to smoother gradients and often faster convergence.

ParameterTypeDescription
y_true[]TLabels in {-1, +1}
y_pred[]TRaw decision scores

Returns f64 — mean squared hinge loss.

fn kl_divergence_loss(y_true []f64, y_pred []f64) f64

Kullback-Leibler divergence: measures how y_pred differs from the reference distribution y_true. Computes sum(y_true[i] * log(y_true[i] / y_pred[i])) for positive y_true entries. Both inputs should be valid probability distributions (non-negative, summing to 1). Predictions are clipped to avoid log(0).

ParameterTypeDescription
y_true[]f64Reference (true) probability distribution
y_pred[]f64Approximate (predicted) probability distribution

Returns f64 — KL divergence (non-negative; zero when distributions are identical).

fn cosine_similarity_loss[T](y_true []T, y_pred []T) f64

Cosine similarity loss: 1 - cosine_similarity(y_true, y_pred). A value of 0 means the vectors point in the same direction; 2 means opposite directions. Useful for embedding and retrieval tasks where direction matters more than magnitude.

ParameterTypeDescription
y_true[]TReference vector
y_pred[]TPredicted vector

Returns f64 — value in [0, 2]; lower is better.

fn contrastive_loss(y_true f64, distance f64, margin f64) f64

Contrastive loss for Siamese networks. For similar pairs (y_true > 0.5) it returns distance^2, pulling the representations together. For dissimilar pairs it returns max(0, margin - distance)^2, pushing them apart only when they are closer than margin.

ParameterTypeDescription
y_truef64Similarity label: 1.0 for a matching pair, 0.0 for a non-matching pair
distancef64Euclidean distance between the two embeddings
marginf64Minimum desired distance between dissimilar pairs; typically 1.0

Returns f64 — contrastive loss for a single pair.

fn triplet_loss[T](anchor []T, positive []T, negative []T, margin f64) f64

Triplet loss for metric learning: max(0, d(anchor, positive) - d(anchor, negative) + margin) where d is Euclidean distance. Encourages the model to place positive examples (same class as anchor) closer than negative examples (different class) by at least margin.

ParameterTypeDescription
anchor[]TReference embedding vector
positive[]TEmbedding from the same class as anchor
negative[]TEmbedding from a different class than anchor
marginf64Minimum separation required between positive and negative distances; typically 1.0

Returns f64 — triplet loss (zero when the constraint is already satisfied by more than margin).

Examples

Binary classification with a 3-layer network

import vstats.nn

fn main() {
    // XOR-like dataset: label is 1 when exactly one of the first two features is high
    x_train := [
        [0.05, 0.05, 0.5],
        [0.95, 0.05, 0.5],
        [0.05, 0.95, 0.5],
        [0.95, 0.95, 0.5],
    ]
    y_train := [0.0, 1.0, 1.0, 0.0]

    mut net := nn.sequential([3, 8, 4, 1], 'relu')
    config := nn.training_config(0.05, 500, 4)
    net.train(x_train, y_train, config)

    preds := net.predict(x_train)
    for i, p in preds {
        println('sample ${i}: pred=${p[0]:.3f}  label=${y_train[i]}')
    }
    mse := net.evaluate(x_train, y_train)
    println('MSE: ${mse:.4f}')
}

sample 1: pred=0.961  label=1.0

sample 3: pred=0.039  label=0.0
MSE: 0.0024

Comparing loss functions for regression

import vstats.nn

fn main() {
    // Predictions with one large outlier
    y_true := [1.0, 2.0, 3.0, 4.0, 100.0]
    y_pred := [1.1, 1.9, 3.2, 3.8,  50.0]

    mse   := nn.mse_loss(y_true, y_pred)
    mae   := nn.mae_loss(y_true, y_pred)
    huber := nn.huber_loss(y_true, y_pred, 1.0)

    println('MSE:   ${mse:.4f}')    // dominated by the outlier
    println('MAE:   ${mae:.4f}')    // less sensitive
    println('Huber: ${huber:.4f}')  // balanced

    // MSE gradient tells the network to focus heavily on the outlier
    mse_grad := nn.mse_loss_gradient(y_true, y_pred)
    println('MSE gradient: ${mse_grad}')
}
Output:

Huber: 49.5750
MSE gradient: [-0.04, 0.04, -0.08, 0.08, -20.0]

1D convolution and pooling on a signal

import vstats.nn

fn main() {
    // Synthetic 1D signal with a spike
    signal := [0.1, 0.2, 0.8, 0.9, 0.8, 0.2, 0.1, 0.0, 0.1, 0.2]

    // Edge-detection kernel (discrete derivative)
    kernel := [-1.0, 0.0, 1.0]

    convolved := nn.conv1d(signal, kernel, 1)
    println('Convolved (stride=1): ${convolved}')
    // [0.1, 0.6, 0.1, -0.1, -0.6, -0.1, 0.1, 0.1]

    // Downsample with max pooling (window=2, stride=2)
    pooled_max := nn.max_pool1d(convolved, 2, 2)
    println('Max pooled (2,2):     ${pooled_max}')
    // [0.6, 0.1, -0.1, 0.1]

    // Smooth with average pooling
    pooled_avg := nn.avg_pool1d(convolved, 2, 2)
    println('Avg pooled (2,2):     ${pooled_avg}')
    // [0.35, 0.0, -0.35, 0.1]

    // Apply softmax to pooled scores to get a probability distribution
    probs := nn.softmax(pooled_max)
    println('Softmax of max pool:  ${probs}')
    // [0.5537, 0.2248, 0.0969, 0.1246]
}
