# Design: chart-gallery example

**Date:** 2026-06-07
**Status:** Approved (design phase)
**Scope:** A single, comprehensive example under `examples/` that showcases the `chart` module by visualizing real regression-analysis outputs.

## Concept

A **regression diagnostics gallery**: one dataset (Boston Housing), one analysis, four charts that together demonstrate every chart type and key feature of the `chart` module. The framing is "charting as the last mile of real vstats work" — every chart renders an actual analysis output, not a disconnected demo.

## Location & files

New directory `examples/chart-gallery/` containing:

- `main.v` — the runnable example. Follows the existing example header convention (a `// Scenario:` / `// Demonstrates:` / `// Python equivalent:` comment block, `module main`, `import vstats.<module>`), and prints a short narration with `println`.
- `README.md` — follows the existing example README format (title, one-paragraph description, a **Verify** line, **Modules used**, **Python equivalent**).
- Four committed `.svg` reference outputs (generated by running the example): `regression_fit.svg`, `residuals_vs_fitted.svg`, `residuals_hist.svg`, `coefficients.svg`.

## The four charts

Each chart renders a real output of the regression analysis.

| File | Type | Analysis | Features demonstrated |
|------|------|----------|-----------------------|
| `regression_fit.svg` | scatter + line | Simple linear regression of median home price on the rooms feature (`RM`): raw data points plus the fitted line (over sorted x) | multi-series, legend, title, x/y axis labels |
| `residuals_vs_fitted.svg` | scatter + guide | Residuals (`y - y_pred`) plotted against fitted values | `axhline(0.0)` zero-reference guide line |
| `residuals_hist.svg` | histogram | Distribution of the residuals, automatic bin count (Sturges) | histogram, auto-binning |
| `coefficients.svg` | bar | Coefficients of a full multivariate regression on all features | bar chart plus a custom `Theme` override (grid enabled) |

**Coverage:** line, scatter, bar, histogram (all four types); legend, `axhline` guide, title and axis labels, custom theme override, multiple series on one chart, and save-to-disk.

## Data flow

1. `dataset := utils.load_boston_housing()!` — features `[][]f64`, target `[]f64`, feature names `[]string`.
2. Extract the `RM` (average rooms) column as a single-feature matrix; fit `ml.linear_regression` on it. Produce predictions with `ml.linear_predict`. Compute residuals as `target[i] - pred[i]`.
   - Chart 1: scatter of `(RM, price)` plus a line of `(RM_sorted, pred_sorted)`.
   - Chart 2: scatter of `(pred, residual)` with `axhline(0.0)`.
   - Chart 3: histogram of residuals.
3. Fit a second `ml.linear_regression` on the full feature matrix; read `model.coefficients`.
   - Chart 4: bar of the coefficients (one bar per feature) with a grid-enabled theme.
4. stdout narration: dataset size, simple-model RMSE (`ml.rmse`), intercept, and the strongest (largest absolute) coefficient with its feature name; finally `wrote 4 charts to examples/chart-gallery/`.

## Error handling

`fn main()` uses `!`-propagation on `utils.load_boston_housing()!` and on each `chart....save(path)!`, matching the existing examples (an error panics the example). Output paths are built relative to the example directory so the committed `.svg` files land beside `main.v`.

## Verification

This example is fast and file-producing (unlike the slower ML examples marked "do not run"). The implementation plan will actually run it:

- `v run examples/chart-gallery/main.v` exits 0 and prints the narration.
- The four `.svg` files exist under `examples/chart-gallery/` and each begins with `<svg` and ends with `</svg>`.

## Docs wiring

Add a section for this scenario in `docs/src/examples.md` with an `<!-- include: examples/chart-gallery/main.v -->` tag, consistent with the existing seven examples. The README's **Modules used** lists `vstats.chart`, `vstats.ml`, `vstats.utils`.

## Out of scope

- No unit tests for the example (examples are verified by running/`v -check`, not unit-tested).
- No new `chart` module features — the example uses only the existing v1 API.
- No raster (PNG/JPG) output.
